# AEL v0.2.2 · Track C — Brand Cleanup & Persona Hygiene

CEO autonomous-loop goal file for **Track C** of the v0.2.2 sprint.
Runs in parallel with Track A. Does not depend on Track B.

## Mission

The AEL brand is supposed to be the user's only mental model. The
underlying aiplus substrate is supposed to be invisible. In practice,
the substrate name leaks into user-facing surfaces in several places:

1. PI persona §3.1 emits dispatch commands as `aiplus agent route ...`
   — so when PI tells you what to run next, it says `aiplus` instead
   of `ael`.
2. README + README.zh-CN FAQ entries still mention `aiplus uninstall`,
   `aiplus doctor`, `aiplus install <runtime>` as commands to type.
3. The persona-behavior tests are LLM-driven and occasionally produce
   false negatives (the coauthor-liaison test once said "Approved."
   which is a forbidden substring). No retry mechanism — a single
   flake fails CI.

Track C closes all three.

## Scope (3 bundles, C1 / C2 / C3)

### C1. PI persona §3.1 — substrate-flavored examples → AEL-flavored

`core/templates/personas/pi.md` section 3.1 ("Turning narrative
dispatch into a real artifact") instructs PI to emit shell commands
the user can run to commit a dispatch. The example commands currently
look like:

```bash
aiplus agent route theorist  "identification check..."
aiplus agent route ra-stata  "implement prefecture-pair FE..."
aiplus agent route --workflow author-critic-fixer writer  "draft rebuttal..."
```

These leak the substrate name into PI's chat output. Users see PI tell
them "run `aiplus agent route ...`" and wonder what aiplus is.

Acceptance criteria:

- C1.1. All example dispatch commands in pi.md §3.1 use `ael route ...`
  (the AEL wrapper shipped in PR #69+).
- C1.2. The `--workflow author-critic-fixer` example uses
  `ael route --workflow author-critic-fixer writer ...`.
- C1.3. The persona-behavior test that exercises PI dispatch emission
  is updated to expect `ael` not `aiplus` in dispatch examples.
- C1.4. Sync target: `vendor/aiplus/assets/aieconlab/core/templates/personas/pi.md`
  is built FROM AEL via `scripts/build-ael.sh` rsync, so the source
  of truth is `core/templates/personas/pi.md` in this repo. No need
  to touch the aiplus vendor side directly.

### C2. README + FAQ substrate-leak sweep

Acceptance criteria:

- C2.1. `grep -rn 'aiplus' README.md README.zh-CN.md` returns ONLY
  lines that are:
  - In a code block clearly marked as "substrate-internal" / "advanced

  - Or in the technical architecture section explaining what AEL is

- C2.2. No FAQ entry or "how to" section instructs the user to type
  `aiplus <something>`. Every instruction is `ael <something>`.
- C2.3. Update the "How do I undo the install?" FAQ entry to refer to
  `ael uninstall --yes` (coordinate with Track B who's adding the

- C2.4. Update the "It says NEEDS_FIX" FAQ entry to refer to

### C3. persona-behavior test retry mechanism

`tests/persona-behavior.test.sh` runs LLM-driven role compliance
checks. These tests are inherently stochastic — the LLM sometimes
produces output that fails the assertion when the same prompt would
pass on a re-run.

PR #65 saw this: coauthor-liaison test said "Approved." (forbidden
substring per its persona), failed CI. Re-running the same job
passed.

Acceptance criteria:

- C3.1. Wrap each persona-behavior assertion in a retry-up-to-3
  loop. If the test fails on attempts 1 or 2, log a warning, sleep
  briefly, retry. If attempt 3 fails, the test legitimately fails.
- C3.2. Output clearly distinguishes "failed cleanly" from "passed
  after retry N=2". Append the retry count to CI step summary so
  Owner can spot if tests are becoming chronically flaky.
- C3.3. Per-test budget: each persona-behavior assertion takes ~30s
  on the slow path (LLM call). 3x retry = ~90s worst case per assertion.
  Verify total CI time still under the 20-min job timeout.

## Halt conditions

- S1. C1: when PI persona §3.1 is updated to emit `ael route`, the
  persona-behavior CI may fail because previous expected outputs
  contained `aiplus agent route`. Stop and ask: should we also bulk-
  update the persona-behavior test fixtures, or accept that some
  tests need a fresh baseline?
- S2. C2: some README content is hard to categorize (e.g. the "What
  is AEL built on?" section legitimately names aiplus as the
  substrate). Stop and ask Owner: is this disclosure OK, or rephrase
  to "AEL is a research-team layer on top of an open-source agent
  CLI" with no specific name?
- S3. C3 retry mechanism: if a persona test is failing 3 times in a
  row, that suggests the persona itself drifted, not a flake. Stop
  and don't auto-pass on N=3 — escalate to Owner as "persona
  regression suspected".
