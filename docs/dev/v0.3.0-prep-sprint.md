## Work breakdown

### B1: Memory audit (exploratory, ~45 min)

Goal: characterize current cross-session memory behavior in AEL/aiplus so we know what to optimize in v1.0 vs v1.1.

Run 3 dogfood scenarios in a real research project (use `/tmp/ael-memory-audit-XXXX` or a scratch dir):

**Scenario M1**: Single-role persistence
1. `ael writer` → draft 3 paragraphs about "RD identification" → exit
2. Close terminal, open new one
3. `ael writer` again → ask "what did we write yesterday?"
4. Record: does writer know? Or starts fresh?

**Scenario M2**: Cross-role visibility
1. `ael advisor` → critique a hypothetical "weak instrument" story → exit
2. `ael writer` → ask "advisor flagged something about IV — what was it?"
3. Record: does writer have access to advisor's last critique?

**Scenario M3**: Long-gap continuity
1. `ael pi` → dispatch a task to ra-stata → exit
2. Wait 5+ minutes
3. `ael pi` again → ask "what was the last task I dispatched?"
4. Record: does PI remember the dispatch?

For each scenario, capture:
- Actual behavior (verbatim where useful)
- Memory artifacts on disk (`.aiplus/agent-memory/*`, `~/.aiplus/memory/*`, runtime session files)
- Whether persona prompt auto-loaded memory or started fresh

Write findings to `docs/dev/_archive/v0.3.0-memory-audit.md` (~60-80 lines). Highlight 3 things:
1. What works today (don't break in v1.0)
2. Gaps (catalog for v1.1 backlog)
3. Any quick wins worth doing in v1.0 (low-effort, high-impact)

### B2: AEL `ael <role>` default-resume + env var wiring (~45 min)

Per v1.0 design Section 1 row 2 + 8 + 9: rewire `ael` bash entry to use aiplus shipped protocols.

**Changes to `ael`** (Mac/Linux bash entry):

1. Near top of file (after `AEL_VERSION` block, before any commands):
   ```bash
   # Use aiplus v0.6.16+ env vars to brand + select team
   export AIPLUS_BRAND="${AIPLUS_BRAND:-AEL}"
   export AIPLUS_TEAM="${AIPLUS_TEAM:-aieconlab}"
   ```

2. In `shortcut_talk_cmd` (the function that maps `ael pi` → `aiplus agent talk pi`): add `--resume` by default unless `--fresh` is passed.

   ```bash
   # If user passed --fresh, strip it and don't add --resume
   # Otherwise, default to --resume (aiplus v0.6.16+ supports this)
   ```

3. **Delete** `ael_bypass_enabled()` function and all its callers. aiplus v0.6.18 is global default-on; AEL no longer needs to inject `--bypass`.

4. **Delete** related env var refs: `AEL_BYPASS` becomes dead (users should use `AIPLUS_SAFE=1` if they want safe mode). Document this in CHANGELOG + a one-line `AEL_BYPASS` removal note in README.

5. Bump `AEL_VERSION` to `0.3.0`. Also bump `scripts/build-ael.sh`.

**Changes to `bin/ael.ps1`** (Windows): mirror the env var exports + `--resume` default + bypass deletion.

**Changes to tests**: update `tests/ael_wrapper.test.sh` for new behavior. Drop the bypass-injection tests, add resume-default tests.

**Spike validation** (CEO runs after impl):
```bash

AIPLUS_BRAND= ael status   # should still work (default kicks in)

```
