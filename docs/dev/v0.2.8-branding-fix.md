# AEL v0.2.8 · "AiPlus virtual team" → team-aware brand — CEO autonomous goal

Companion to the short `/goal` text. CEO reads this fully before
starting.

## Mission

When ael opens a runtime interactively, the first line of the prompt
sent to the runtime currently reads:

> You are the `pi` of the **AiPlus virtual team** installed in this
> project.

Owner dogfooded v0.2.7 (2026-05-19, ~22:50) on the Qing_Ideology
project. Both `ael pi` (codex) and `ael advisor` (claude-code)
sessions worked end-to-end — bypass mode active, identity context
PASS, persona loaded — but the wrapper sentence leaked the upstream
substrate's brand name ("AiPlus") into the agent's worldview.

This is the last visible piece of substrate-brand leakage in the AEL
interactive UX. ael's stdout sanitizer (perl regex in `ael`,
`s/\bAiPlus\b/AEL/g`) catches substrate stdout but does NOT catch
this string, because the wrapper sentence is part of the prompt
sent by aiplus directly into the runtime's stdin (via exec — never
flows through ael's pipe).

## Root cause (already located)

Source file:
`vendor/aiplus/crates/aiplus-cli/src/agent/talk.rs`

Function: `build_talk_prompt(role: &str, persona: &str) -> String`
at lines 249-261.

Current code (line 251):

```rust
"You are the `{role}` of the AiPlus virtual team installed in this project. \
```

The string is hardcoded. There is no team-awareness, even though
the same file's caller (`handle_talk`) already calls
`resolve_role_for_active_team(...)` which consults the team marker
file. The helper `crate::agent::set_team::read_active_team(project_root)
-> Option<String>` returns "aieconlab" / "agent-team" / None.

## Architectural choice

Make `build_talk_prompt` team-brand-aware. Two reasonable shapes;
**A is preferred** for clarity.

### Option A — explicit `team_brand: &str` parameter (chosen)

```rust
fn build_talk_prompt(role: &str, persona: &str, team_brand: &str) -> String {
    format!(
        "You are the `{role}` of the {team_brand} virtual team installed in this project. \
        ...",
    )
}
```

`handle_talk` resolves the brand once:

```rust
let active_team = crate::agent::set_team::read_active_team(&project_root);
let team_brand = team_brand_for(active_team.as_deref());
let prompt = build_talk_prompt(canonical_role, &persona_text, team_brand);
```

with helper:

```rust
fn team_brand_for(active_team: Option<&str>) -> &'static str {
    match active_team {
        Some("aieconlab") => "AEL",
        _ => "AiPlus", // default for agent-team or unknown
    }
}
```

### Option B — global `BrandMap` lookup (rejected)

Heavier (config, registry, lifetime concerns) for one string. Skip.

## Scope (1 PR in aiplus, 1 PR in AEL)

### B1. aiplus substrate (separate repo: `izhiwen/AiPlus`)

This is in `izhiwen/AiPlus`, **not AEL**. CEO of AEL does NOT touch
vendor/aiplus directly — open the change in the AiPlus repo, get it
merged + tagged, then bump the vendor pin in AEL.

Acceptance criteria:

- B1.1. `build_talk_prompt` gains a `team_brand: &str` parameter; the
  hardcoded "AiPlus" in the wrapper sentence becomes `{team_brand}`.
- B1.2. New private helper `team_brand_for(active_team: Option<&str>)
  -> &'static str` returns "AEL" for `Some("aieconlab")`, "AiPlus"
  for everything else (including `None` and `Some("agent-team")`).
  Co-locate in `talk.rs` — do not add a new module.
- B1.3. `handle_talk` calls `read_active_team(&project_root)` ONCE,
  computes `team_brand`, passes into `build_talk_prompt`. Do not
  read the marker file twice.
- B1.4. Unit tests in `talk.rs`:
  - `team_brand_for(Some("aieconlab")) == "AEL"`
  - `team_brand_for(Some("agent-team")) == "AiPlus"`
  - `team_brand_for(None) == "AiPlus"`
  - `team_brand_for(Some("unknown")) == "AiPlus"`
  - A `build_talk_prompt(role, persona, "AEL")` snapshot test that
    asserts the output begins with "You are the `<role>` of the AEL
    virtual team installed in this project."
- B1.5. No CHANGELOG-level wording change to the wrapper sentence
  beyond the brand variable. Do NOT take this opportunity to also
  reword "installed in this project" or anything else — surgical.
- B1.6. Bump aiplus-cli patch version (v0.6.14). Tag + release —
  this is a substrate behavior change visible to all wrapper users.
- B1.7. Title: `feat(agent): team-aware talk brand in build_talk_prompt`.
