# AEL v0.2.6 · CI repair — micro-sprint

CEO autonomous-loop goal file. Companion to the short `/goal` text.

## Mission

The v0.2.4 sprint (commits `bde35fd` + `e410bfa`) shipped two new
test sections in `tests/ael_wrapper.test.sh` (the "install all" path
test and the "shortcut --runtime forwarding" test). Both PASS on a
developer Mac that happens to have `claude` and `codex` CLIs in
PATH. Both FAIL on the GitHub Actions ubuntu-latest runner, which
has no AI runtime CLIs installed.

Symptom on CI:

```
Running tests/ael_wrapper.test.sh
ael: claude CLI not found on PATH
##[error]Process completed with exit code 1.
```

Every PR opened after 2026-05-19 21:51 UTC has a red `validate-assets`
check. Even docs-only PRs (e.g. #116 for v0.2.5 goal) fail because
`validate-assets` runs the whole test suite.

This micro-sprint repairs main's CI so subsequent PRs (including the
v0.2.5 sprint) can land cleanly.

## Scope (1 bundle, time-boxed)

### CI1. Repair `tests/ael_wrapper.test.sh` for environments without runtime CLIs

Acceptance criteria:

- CI1.1. Identify the exact assertion(s) in `tests/ael_wrapper.test.sh`
  that invoke wrapper code paths requiring a real CLI binary
  (`claude` / `codex` / `opencode`) on PATH. The "shortcut runtime
  forwarding" test (around line 108-140 on commit e410bfa) is the
  prime suspect, but verify by tracing.
- CI1.2. For each such assertion, fake the missing CLI in a tmpdir
  (same pattern as the existing `$shortcut_runtime_bin/claude`
  setup, which DOES work for that specific test). The fix is
  whichever assertion lacks the fake.
- CI1.3. Alternative: if a code path inherently needs the real CLI
  (rare), guard the test with a "skip if not installed" check that
  exits 0 with a clear message — but prefer faking over skipping.
- CI1.4. After fix, the test must PASS:
  - Locally on Mac with `claude` in PATH (already works)
  - Locally on Mac without `claude` in PATH:
    `PATH="/usr/bin:/bin" bash tests/ael_wrapper.test.sh`
    must pass
  - On GitHub Actions ubuntu-latest validate-assets job
- CI1.5. No new test cases added; this is pure repair. (Adding new
  coverage is v0.2.5's job, not v0.2.6.)
- CI1.6. One PR only. Title prefix: `fix(tests):`.

## Halt conditions

- S1. If the broken assertion turns out to be in `ael` wrapper code
  (not test code) — i.e., the wrapper has new behavior that
  legitimately requires the CLI before any work can begin — halt
  and ask: was that intentional in v0.2.4 design, or a regression?
- S2. If the fix breaks the "shortcut --runtime forwarding" coverage
  (which is what v0.2.4 PR #110 added), halt and ask: should we
  preserve that test, modify it, or temporarily mark xfail?

## Anti-patterns

- Do NOT change `ael` wrapper code. The bug is in test setup, not
  production code.
- Do NOT add a CI matrix entry just to skip the test.
- Do NOT touch unrelated files (no version bumps, no persona
  changes, no docs except a CHANGELOG line if you want).
- Do NOT rewrite the whole test suite. Surgical fix only.

## Done criteria

- D1. One PR merged.
- D2. validate-assets check on a fresh PR (e.g. #116 v0.2.5 goal) is
  GREEN after this PR merges + #116 rebases.
- D3. Sprint summary comment with the failing line numbers + the
  one-line fix that resolved them.
