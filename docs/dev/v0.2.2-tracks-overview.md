## How Owner spawns the CEOs

Each CEO runs in its own terminal with Codex CLI (or whichever runtime
you use for autonomous loops). Concretely:

```bash
# Terminal 1 — Track A
cd ~/Projects/AiEconLab
codex --goal docs/dev/v0.2.2-track-a-release-pipeline.md

# Terminal 2 — Track C  (start same time as Track A)
cd ~/Projects/AiEconLab
codex --goal docs/dev/v0.2.2-track-c-brand-cleanup.md

# Terminal 3 — Track B  (start AFTER A1 PR is merged)
cd ~/Projects/AiEconLab
codex --goal docs/dev/v0.2.2-track-b-ux-commands.md
```

Each terminal becomes a self-contained CEO session. They write to
separate branches (`track-a/*`, `track-b/*`, `track-c/*`) and open PRs
that Owner reviews.
