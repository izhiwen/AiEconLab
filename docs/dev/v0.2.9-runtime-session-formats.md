# v0.2.9 Runtime Session Formats

Discovery date: 2026-05-20

Scope: macOS AEL `talk --resume` support. The implementation must use bash
plus standard text tools only; no `jq`, `fzf`, Python, or non-shell parser is
introduced for session parsing.

## Codex

- CLI checked: `codex-cli 0.131.0`.
- Session root: `~/.codex/sessions/YYYY/MM/DD/`.
- File naming: `rollout-<timestamp>-<uuid>.jsonl`.
- File format: JSONL text.
- Session id: `payload.id` on the `session_meta` line.
- Session timestamp: `payload.timestamp` or the line `timestamp` on
  `session_meta`.
- Session cwd: `payload.cwd` on `session_meta`.
- First user message: first later JSONL message whose payload contains
  user text; AEL filters by the literal role signature `` `<role>` of ``.
- Resume by id: `codex resume <session-id>`.

Status: fully supported. AEL filters JSONL files by exact cwd and role
signature, sorts by timestamp descending, and resumes the chosen session id.

## Claude Code

- CLI checked: `2.1.145 (Claude Code)`.
- Session root: `~/.claude/projects/<encoded-cwd>/`.
- File naming: `<uuid>.jsonl`.
- File format: JSONL text.
- Session id: filename without `.jsonl`.
- Session timestamp: `timestamp` on the user message line, falling back to the
  first timestamp in the file.
- Session cwd: `cwd` field on recorded entries.
- First user message: first user entry containing prompt text; AEL filters by
  the literal role signature `` `<role>` of ``.
- Resume by id: `claude --resume <session-id>`.

Status: fully supported. AEL scans project JSONL files by exact cwd and role
signature, sorts by timestamp descending, and resumes the chosen session id.

## OpenCode

- CLI checked: `1.15.3`.
- Session metadata root: `~/.local/share/opencode/opencode.db`.
- Related data roots:
  - `~/.local/share/opencode/storage/session_diff/*.json`
  - `~/.local/share/opencode/snapshot/`
- File/database format: SQLite for authoritative session, message, part,
  project, and workspace tables. Snapshot and diff files are not sufficient to
  recover first user message, cwd, timestamp, and session id together.
- Useful tables observed:
  - `session(id, directory, title, time_created, time_updated, ...)`
  - `message(id, session_id, data, time_created, ...)`
  - `part(id, message_id, session_id, data, time_created, ...)`
- Resume by id: `opencode --session <session-id>` or `opencode run --session
  <session-id>`.
- Last-session resume: `opencode --continue`.
- Native listing: `opencode session list`.

Status: degraded by S1. OpenCode has session-id resume, but role-aware
enumeration requires SQLite parsing. v0.2.9 does not add a SQLite dependency,
so `ael talk --resume <role>` falls back to OpenCode's native UI,
`--last` falls back to `opencode --continue`, and `--list` prints
`opencode session list`.
