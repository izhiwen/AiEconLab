# AEL v0.2.10 — Lobby Delegate to aiplus

**Drafted**: 2026-05-20 by aiplus Advisor (cross-repo coordination)
**Status**: PROPOSAL — awaits Owner approval → AEL CEO briefing
**Estimated effort**: ~3 hr CEO (bash refactor + test updates)
**Depends on**: aiplus v0.6.16 published ✅ (lobby + `AIPLUS_BRAND` env support shipped)
**Parallel-safe**: zero file overlap with aiplus v0.6.17 observability bundle — both can run simultaneously in different repos.

## Context

aiplus v0.6.16 (shipped 2026-05-20) added:
- `aiplus` (no args) lobby with resume support, brand parameterization
- `AIPLUS_BRAND` / `AIPLUS_TEAM` / `AIPLUS_DEFAULT_ROLE` env vars
- `aiplus agent talk --resume <role>`
- `[BRAND:role:project]` session tag prefix

Today, AEL bash duplicates aiplus lobby UX with `sanitize_substrate_output` perl filter to swap "AiPlus" → "AEL". This duplication:
1. Misses new aiplus features (resume, brand, etc.) until AEL bash is manually updated
2. Is fragile (perl regex substring matching can over-replace)
3. Was the wrong architectural fit (v0.2.9 role-aware picker in bash, since deleted per Owner approval)

## Goal

**Delete AEL's own lobby logic. Replace with `exec env AIPLUS_BRAND=AEL ... aiplus "$@"`.**

After v0.2.10, AEL is:
- AEL-specific commands (install / update / uninstall / doctor / status) → handled by AEL bash
- Everything else (lobby, talk, route, ...) → exec aiplus with env vars set

AEL bash shrinks from 52KB to ~20KB. AEL inherits all future aiplus lobby/talk improvements automatically.

- aiplus side changes (those are in v0.6.17 observability bundle, different repo)

**Delete these functions/sections in `ael`**:

| Function | Approximate lines | Reason |
|---|---|---|

**Keep these (AEL-specific)**:

- `cmd_install`, `cmd_update`, `cmd_uninstall`, `cmd_doctor`, `cmd_status` (handle AEL repo's install/update flow)
- `ensure_project_ready_for_lobby` (verifies `.aiplus/agents/personas/*.md` present)
- aiplus PATH sync logic (`update_sync_aiplus_path`)
- Version banner / `--version`

**Rewrite `main()`** (around line 1499):

```bash
main() {
  case "${1:-}" in
    install|update|uninstall|doctor|status|--version|-v|--help|-h)
      handle_ael_specific "$@"
      ;;
    *)
      # Ensure project is ready (writes personas if missing)
      ensure_project_ready_for_lobby

      # Delegate everything else to aiplus with AEL brand
      exec env \
        AIPLUS_BRAND="AEL" \
        AIPLUS_TEAM="aieconlab" \
        AIPLUS_DEFAULT_ROLE="pi" \
        aiplus "$@"
      ;;
  esac
}
```

That's it. Total bash delta: roughly **-400 lines + 20 lines new main()**.

## Edge cases

### 1. aiplus not on PATH or too old (<v0.6.16)
Detect at AEL startup:
```bash
if ! command -v aiplus >/dev/null 2>&1; then
  die "aiplus not on PATH. Run \`ael install\` to set it up."
fi
aiplus_version=$(aiplus --version 2>/dev/null | head -1 | grep -oE 'v?[0-9]+\.[0-9]+\.[0-9]+')
if ! version_at_least "$aiplus_version" "0.6.16"; then
  die "AEL v0.2.10+ requires aiplus v0.6.16+; found $aiplus_version. Run \`ael update\`."
fi
```

Print clear remediation. Don't try to half-work.

### 2. User has stale .aiplus/team.toml pointing to old team name
`ensure_project_ready_for_lobby` should validate `active_team=aieconlab` or warn:
```
⚠️ This project's .aiplus/team.toml is set to team=<something>, not aieconlab.
   aiplus lobby will not use AEL personas correctly.
   Run `ael install` to repair, or edit .aiplus/team.toml manually.
```

### 3. v0.2.9 picker users with active resume sessions
Sessions tagged by v0.2.9's bash picker may have different title format than aiplus v0.6.16's `[BRAND:role:project]` convention. They show as `[?]` in aiplus lobby — same fallback as pre-v0.6.16 sessions per aiplus spec. Non-blocking; sessions still resume by mtime.

### 4. AEL on Windows (if supported)
AEL bash via Git Bash. `exec env ...` works in Git Bash. Verify in tests.

### 5. AEL_VERSION env var override
Existing `AEL_VERSION="${AEL_VERSION:-0.2.10}"` pattern continues to work for testing.
