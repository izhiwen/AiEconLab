# AEL v0.2.10 Implementation Notes

Date: 2026-05-20

## Scope

- Refactored `ael` from a local lobby/runtime/resume wrapper into a thin
  AEL maintenance wrapper plus aiplus delegate.
- Removed the bash-owned v0.2.9 lobby picker, runtime picker, role-aware
  resume picker, headless talk path, and substrate output sanitizer.
- Kept AEL-specific maintenance commands in bash:
  `install`, `update`, `uninstall`, `doctor`, `status`, `--version`, and
  `--help`.

## Delegate Contract

Delegated commands exec `aiplus` with:

```bash
AIPLUS_BRAND=AEL
AIPLUS_TEAM=aieconlab
AIPLUS_DEFAULT_ROLE=pi
```

Mappings retained for AEL user ergonomics:

- `ael` -> `aiplus`
- `ael <role>` -> `aiplus agent talk <role>`
- `ael talk ...` -> `aiplus agent talk ...`
- `ael route ...` -> `aiplus agent route ...`
- `ael invite|dismiss|integrate ...` -> `aiplus agent <subcommand> ...`
- Other non-AEL-specific commands pass through unchanged.

## Preconditions

- `aiplus --version` on this machine returned `0.6.13`, not the required
  `0.6.16+`. Delegate-path tests therefore use a shim for v0.6.16.
- The working tree was not clean at start because
  `docs/dev/v0.2.10-lobby-delegate.md` was present as an untracked
  authoritative spec file.
