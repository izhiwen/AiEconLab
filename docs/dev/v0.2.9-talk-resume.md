# v0.2.9 sprint goal — `ael talk --resume <role>` (role-aware session picker)

**Status**: SPEC — awaits CEO execution
**Owner**: codex CEO autonomous loop
**Estimated effort**: 2–3 hours single-pass
**Depends on**: v0.2.8 (branding fix) merged to main

---

## Problem

`ael talk <role>` spawns a runtime (codex / claude-code / opencode) via
`aiplus agent talk`. When the runtime dies — OOM, network drop,
accidental Ctrl-C, runtime upgrade — the AEL user has no built-in
recovery. They must remember which runtime that role uses, cd to the
project, and invoke the runtime's native resume command.

That breaks AEL's "one command does it" promise exactly when the user
is mid-research and stressed.

## Goal

Single command:

```bash
ael talk --resume advisor
```

Behavior:
1. Detect which runtime that role uses (reuse `read_active_team` +
   team config — same logic as fresh `ael talk`).
2. Enumerate that runtime's session history.
3. Filter by **(a)** current project directory, **(b)** AEL role
   signature in first user message: regex `\`<role>\` of`.
4. Show numbered picker (timestamp + first-message preview).
5. User picks → exec runtime to resume that session by id.

Convenience modes:
- `ael talk --resume <role> --last` — skip picker, take most recent.
- `ael talk --resume <role> --list` — print list and exit (scripting).

## Why role-aware (not thin runtime delegation)

Researchers accumulate many runtime sessions per project across roles
(advisor / pi / ra / referee / replicator). Runtime-native pickers
show ALL sessions, polluting the choice. The role filter cuts noise
so crash recovery is fast and confident — which matters because
users invoke this exactly when something just broke.

## Scope — B1 (single-repo, AEL only)

No aiplus changes. All work in the `ael` bash wrapper.

### B1.0 — Discovery (do this first, ~30 min)

For each runtime, document:
- Session storage directory (e.g., `~/.codex/sessions/`,
  `~/.claude/projects/<encoded-cwd>/`, opencode TBD).
- File naming convention (uuid? timestamp?).
- File format (JSONL? sqlite? plain text?).
- How to extract: **(a)** session timestamp, **(b)** cwd of session,
  **(c)** first user message (we grep first ~500 chars for role
  signature).
- How to invoke "resume session X" by id.

Write findings into `docs/dev/v0.2.9-runtime-session-formats.md` for
future maintainers. This doc lives in the same PR as the impl.

If any runtime has no session-by-id resume (e.g., opencode might lack
it), document the degraded mode — see halt condition S2.

### B1.1 — Helper functions in `ael`

New shell functions (sketch):

```bash
list_role_sessions() {
  local runtime="$1" role="$2" project_root="$3"
  case "$runtime" in
    codex)       _list_codex_sessions       "$role" "$project_root" ;;
    claude-code) _list_claude_sessions      "$role" "$project_root" ;;
    opencode)    _list_opencode_sessions    "$role" "$project_root" ;;
  esac
}

resume_role_session() {
  local runtime="$1" session_id="$2"
  case "$runtime" in
    codex)       codex resume "$session_id" ;;
    claude-code) claude --resume "$session_id" ;;
    opencode)    opencode resume "$session_id" ;;  # confirm in discovery
  esac
}
```

Output of `list_role_sessions` is tab-separated:

```
<unix-timestamp>\t<session-id>\t<first-message-preview-truncated>
```

Sort by timestamp desc. Truncate preview to ~70 chars.

### B1.2 — Picker UI

Bash builtins only. No `fzf`, no `jq`. Hand-rolled numbered list +
`read`:

```
Recent advisor sessions in this project (4 found):

  1. 2026-05-19 22:14   "Let's review the identification strategy..."
  2. 2026-05-18 14:02   "I want to push back on the parallel trends..."
  3. 2026-05-15 09:33   "Looking at the IV first stage F-stat..."
  4. 2026-05-12 18:45   "Initial draft scope check..."

Pick (1-4, q to quit) [1]: _
```

Default to 1 on bare Enter (most recent). `q` exits with code 130.

### B1.3 — Flag parsing

In `ael`'s `talk` subcommand argument parser, add:
- `--resume` (turns on resume mode)
- `--last` (only valid with `--resume`; skip picker)
- `--list` (only valid with `--resume`; print and exit)

Existing fresh-talk path unchanged when `--resume` absent.

### B1.4 — Empty-state behavior

Zero matching sessions: print

```
No advisor sessions found in this project. Start a fresh one:

```

Exit code: 2 (so scripts can detect "no match" vs "user cancelled").

## Halt conditions

**S1 — Unreliable parse**: A runtime stores sessions in a format that
cannot be reliably parsed with bash + grep (e.g., binary sqlite).
Halt B1 implementation for that runtime; degrade to runtime-native
resume (`<runtime> resume` with no id, show runtime's own TUI).
Document the degraded mode in B1.0 findings. Continue with the other
runtimes.

**S2 — No session-id resume**: A runtime has no way to resume a
specific session by id (only "resume last"). Halt; degrade to
runtime-native resume. Document and continue.

**S3 — Signature collision**: First-message role signature is
unreliable (e.g., role name appears in unrelated sessions, persona
prompt structure too variable). Halt and write findings to
`docs/dev/v0.2.9-signature-findings.md`, ping advisor. Likely needs a
B2 (aiplus emits a session-tag file alongside) — that becomes a
cross-repo sprint and is out of scope for v0.2.9.

If two of three runtimes hit S1/S2, still ship — degraded modes are
acceptable. If all three hit, halt the sprint and ping advisor.

## Anti-patterns — DO NOT do these

- Do NOT add `jq` or `fzf` as runtime dependencies. Bash only.
- Do NOT modify aiplus persona prompts to make grepping easier. Use
  what v0.2.8 produces as-is.
- Do NOT break the fresh-talk path. `ael talk <role>` with no
  `--resume` must behave identically to v0.2.8.
- Do NOT shell out to Python or any non-shell tool for session parsing.
- Do NOT cache session lists. Re-enumerate every call. Performance is
  fine (typical user has <50 sessions per runtime).
- Do NOT auto-pick "most recent" silently when multiple matches exist.
  If user didn't pass `--last`, show the picker.
- Do NOT touch Windows. Mac-only for v0.2.9; Windows shim is a future
  sprint.
- Do NOT add `--resume` to `ael talk-as` or any other subcommand. Only
  `ael talk` for now.
